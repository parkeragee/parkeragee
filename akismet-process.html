Possible process

We could verify new contacts with the Akismet API. If the are verified, then we procede like normal with the correct member_status, triggers, notifications, etc. If the are not verified via the Akismet API, then we simply add the contact to the audience with a new member_status of s for spam. We treat s member statuses similar to optout? We don’t add them to mailings, etc. No notifications, triggers, etc. are initiated. We simply add a new tab in Audience for Spam contacts and the Emma user can go in and update the member status if they deem it worthy.

This process is similar to how Netlify recently started removing spam form submissions on customer sites. Read about that here.
